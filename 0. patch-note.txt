// ================================= 파일 기능 ================================== //
- abcdef.html = 뼈대

- styles.css = 꾸미기

- render.js: 화면을 “그리는 엔진”
한 마디로 현재 state(검색/필터/페이지/카테고리)에 맞춰 DOM을 다시 그려주는 렌더러
필터링 결과 가져오기
카테고리별로 묶어서 섹션 렌더
페이지네이션 계산/렌더
렌더 최적화
📌 결론: “DOM 변경은 render.js 한 곳에서만” 이게 핵심 원칙.

- tabs.js: “사용자 입력 → 상태 변경” 연결하는 컨트롤러
tabs.js는 렌더를 직접 그리기보다, 탭/필터 UI 버튼을 눌렀을 때 state를 어떻게 바꾸고, 어떤 흐름으로 렌더를 호출할지를 담당
카테고리 탭 클릭 처리
“전체 보기 / 필터 초기화” 같은 버튼 처리
전환 애니메이션과 연동
📌 결론: tabs.js는 “클릭 이벤트 처리 + state 변경 + 렌더 호출의 규칙” 담당.

- ui-transition.js: 깜빡거림/튀는 애니메이션을 “잠깐 꺼주는” 스위치

- js/ui/toast.js

목적: 앱 어디서든 호출 가능한 “알림 토스트” 기능 제공
페이지에 #toastHost 컨테이너를 1회 생성(없으면 생성)
토스트 요청을 큐(Queue) 로 쌓아서 한 개씩 순서대로 표시
window.showToast(message, type, duration) 전역 API 제공
type: info / success / warn / error
duration: 기본 3000ms
사용되는 곳 예
“저장됨”, “복사됨”, “에러 발생” 같은 UX 피드백

- js/utils.js

목적: 여러 파일에서 공통으로 쓰는 “기본 유틸 함수” 모음
escapeHtml(str)
텍스트를 안전하게 HTML에 출력하기 위해 특수문자 escape
debounce(fn, wait)
검색 입력처럼 연속 이벤트를 묶어서 마지막 1번만 실행
memoryManager.timerManager가 있으면 그걸 쓰고, 없으면 기본 setTimeout 사용
getChosung(str)
한글 초성 추출(초성검색/정렬/검색 보조에 사용 가능)
전역으로 노출하는 것
window.escapeHtml
window.debounce
window.getChosung

- js/ui/theme.js

목적: 테마/디자인 옵션을 실제 DOM에 적용하는 “스타일 적용 엔진”
initializeTheme()
OS 다크모드(prefers-color-scheme) 감지해서 자동 적용
사용자가 직접 테마를 고르지 않은 경우에만 시스템 변경을 따라감
applyTheme(mode)
light / dark / system 적용
localStorage 저장/삭제
다크 토글 버튼 텍스트도 같이 동기화
applyFontSize(mode)
documentElement.dataset.fontSize 설정 + localStorage 저장
applyAnimation(mode)
documentElement.dataset.anim 설정 + localStorage 저장
applyRadius(mode)
documentElement.dataset.radius 설정 + localStorage 저장
전역 API
window.initializeTheme
window.applyTheme
window.applyFontSize
window.applyAnimation
window.applyRadius

- js/ui/settings.js

목적: 설정 패널 UI(열기/닫기/버튼 클릭)를 담당하는 “컨트롤러”

setupSettingsPanel()로 설정 패널을 초기화
설정 FAB 버튼 클릭 → 패널 열기
닫기 버튼/바깥 클릭/ESC → 패널 닫기
저장된 설정값(localStorage)을 읽어서
theme.js의 applyTheme/applyFontSize/applyAnimation/applyRadius 호출로 실제 적용
버튼 active 상태도 같이 동기화
[data-theme], [data-font], [data-anim], [data-radius] 버튼 클릭 이벤트 바인딩
memoryManager.eventManager가 있으면 그걸 쓰고, 없으면 addEventListener로 fallback
전역 API
window.setupSettingsPanel
